## 版本规范

### 版本号格式

- 主版本号.次版本号.修订号 (例如: 0.1.0)
- 主版本号：重大更新，可能包含不兼容的 API 修改
- 次版本号：功能更新，向下兼容
- 修订号：问题修复，向下兼容

### 更新日志格式

- 新增：新功能
- 优化：功能改进
- 修复：问题修复
- 移除：移除的功能
- 废弃：即将移除的功能

### 提交规范

- feat: 新功能
- fix: 修复问题
- docs: 文档修改
- style: 代码格式修改
- refactor: 代码重构
- perf: 性能优化
- test: 测试相关
- chore: 其他修改
